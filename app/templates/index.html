Hur fungerar verktyget?

Verktyget jämför workflow-metadata mellan Helix/AR System-miljöer med en lokal, komplett cache.

ServerkonfigurationAnslutningar och credentials ligger endast i serverns konfiguration. URL, användarnamn och lösenord skickas inte till browsern.
CacheVid pod-start kontrolleras cache automatiskt. Saknas cache byggs den upp, annars körs ändringskontroll. Cache lagras på PVC.
OmbyggnadRensa PVC och starta om podden om du vill bygga om allt från början.
JämförelseVerktyget använder komplett/djup cache. Profil: {{ deep_profile }}. Parallella metadatahämtningar: {{ related_fetch_concurrency }}.
SynkmodellMigrator-lik modell: oförändrade objekt återanvänds och ändrade objekt djupanalyseras.
Cache-scopeFormulär som läses in styrs av cache_scope i hlx-diff.yaml. Workflow filtreras via mapping till de formulär som ingår innan djupmetadata hämtas.
Aktiv include{% if cache_scope.include_form_prefixes %}{{ cache_scope.include_form_prefixes | join(', ') }}{% else %}Alla formulär{% endif %}
Aktiv exclude{% if cache_scope.exclude_form_prefixes %}{{ cache_scope.exclude_form_prefixes | join(', ') }}{% else %}Inga undantag{% endif %}
Scope-läge{% if cache_scope.include_form_prefixes or cache_scope.exclude_form_prefixes %}Strikt formulärscope. Läser Forms, Active Links, Filters, Escalations och försöker härleda relevanta Guides via workflow-referenser. Globala objekt som Menus, Applications, Packing Lists och Web Services cacheas alltid eftersom de inte är direkt formulärkopplade.{% else %}Ingen prefixbegränsning aktiv. Alla konfigurerade objekttyper kan cacheas.{% endif %}
Manuell kontrollKnappen Kontrollera ändringar per miljö kör en inkrementell kontroll mot cachen.
